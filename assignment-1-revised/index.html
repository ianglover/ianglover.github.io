Committing New Changes to Git & Github

When you are ready to update your Github Pages with your new content, open the Github Desktop application and navigate to the studentname.github.io repository.

On the left hand side, you will see all the files which have had changes. You can click any of those files to see what changes have been made - any line of text with a red background has been removed, and any line of text with a green background has been added.

Use the Checkboxes next to file names to select which files you want to update and upload - most likely, you want to update the entire site so leave them all checked.

Now that we have selected which changes we want to include, we need to describe them. In the lower left-corner you will need to write a summary of your changes, and can add an optional description. When complete, select "Commit to main" to add these changes and describe the codebases current state.

Finally, were going to 'Push' our new code from our Local machine to the Github Pages repository online so it's public!
